1//! Types for cookie state management.
2//!
3//! # Cookie jars
4//!
5//! By default HTTP is a mostly stateless protocol, but using cookies allow a
6//! server to request a client to persist specific state between requests. Isahc
7//! does not do this by default, but provides support for cookie state using a
8//! _cookie jar_, which can store a list of cookies in memory between requests
9//! and is responsible for keeping track of which cookies belong to which
10//! domains.
11//!
12//! A cookie jar can be used on a per-request basis or for all requests sent via
13//! a particular HTTP client. You can assign a cookie jar to a request or to a
14//! client via the
15//! [`Configurable::cookie_jar`](crate::config::Configurable::cookie_jar)
16//! extension method. If different cookie jars are assigned on both a request
17//! and the client sending the request, the one assigned to the individual
18//! request will take precedence.
19//!
20//! The global default client instance does not have an assigned cookie jar.
21//!
22//! # Availability
23//!
24//! This module is only available when the [`cookies`](index.html#cookies)
25//! feature is enabled.
